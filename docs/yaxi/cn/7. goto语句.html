goto 语句

Yaxi中的goto语句与C语言的goto存在差异。Yaxi中的goto是用于运行其他脚本文件的。有的时候，我们开发的项目不只一个文件，还需要多个文件配合完成，这个时候就需要一个语句来完成这一功能。对于C++，Java这类语言，在运行时跳到别的文件进行运行是不现实的，因为这类语言都是先进行了编译的，所以它们提供的类似功能是引入别的文件。而对于脚本语言，从一个文件跳到另一个文件运行其中的脚本却是可行的。

以下是goto的应用举例。

在同一目录下，创建两个脚本文件script1.yx和script2.yx，分别加入代码：

script1.yx

@print("Hello")

goto "./script2.yx"

@print("-- end --")

script2.yx

@print("Running script2.yx")

运行script1.yx，输出的顺序是：Hello -> Running script2.yx -> -- end --

我们可以看到，goto语句跳到运行另一个脚本文件后，script1.yx是会等待script2.yx运行完毕后，再接着运行的。

goto后接的文件路径若是相对路径，那么这个路径得是相对于当前运行的脚本文件的。例子如下：

假设一个目录结构如下：

projectX/
    |
    `-- script0.yx
    |
    `-- src/
        |
        `-- script1.yx
        |
        `-- dir1/
            |
            `-- script2.yx

如果我们运行script1.yx，那想从script1.yx跳转到别的脚本文件所需的代码如下：

// script0.yx
goto "../script0.yx"

// script2.yx
goto "./dir1/script2.yx"
